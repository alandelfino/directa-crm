# Identidade do Frontend — Directa CRM

Sistema: **Directa CRM**

O frontend deve preservar a arquitetura existente do projeto.

## Regras Gerais

- Preserve a estrutura atual.
- Siga os padrões já usados no projeto.
- Não invente estrutura nova se já existir padrão equivalente.
- Antes de criar arquivo, verifique se já existe arquivo equivalente.
- Antes de alterar arquivo, entenda sua responsabilidade.
- Frontend não deve alterar arquivos de backend.
- Não altere APIs, banco, controllers, services, schemas ou migrations em tarefas frontend.

## Diretriz Geral

Ao implementar qualquer funcionalidade, priorize:

- Reutilização de componentes existentes.
- Consistência com o design system.
- Código modular.
- Arquivos pequenos e organizados.
- Tipagem forte.
- Clareza de responsabilidade.
- Manutenção futura.
- Acessibilidade.
- Responsividade.
- Simplicidade sem perda de qualidade.
